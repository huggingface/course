## اپنی ٹیم کو annotation guidelines کے ساتھ ہم آہنگ کریں

اپنے dataset کی annotations شروع کرنے سے پہلے، یہ ایک اچھا عمل ہے کہ آپ کچھ guidelines لکھیں، خاص طور پر اگر آپ ٹیم کے حصے کے طور پر کام کر رہے ہیں۔ اس سے آپ کو ٹاسک اور مختلف labels کے استعمال پر ہم آہنگی پیدا کرنے میں مدد ملے گی، اور جب سوالات یا تنازعات سامنے آئیں تو انہیں حل کرنے میں آسانی ہوگی۔

Argilla میں، آپ UI میں اپنے dataset کی settings کے صفحے پر جا کر اپنی guidelines اور اپنے سوالات کی تفصیلات کو تبدیل کر سکتے ہیں تاکہ ہم آہنگی حاصل کی جا سکے۔

<img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter10/argilla_dataset_settings.png" alt="Screenshot of the Dataset Settings page in Argilla."/>

اگر آپ اچھے guidelines لکھنے کے موضوع پر مزید گہرائی میں جانا چاہتے ہیں، تو ہم [اس blogpost](https://argilla.io/blog/annotation-guidelines-practices) اور وہاں مذکور bibliographical حوالہ جات پڑھنے کی تجویز دیتے ہیں۔

## ٹاسک کو تقسیم کریں

Dataset settings کے صفحے میں، آپ dataset کی تقسیم کی ترتیبات بھی تبدیل کر سکتے ہیں۔ یہ آپ کو ٹیم کے حصے کے طور پر کام کرتے ہوئے annotations کو مزید مؤثر طریقے سے انجام دینے میں مدد دے گا۔ کم از کم جمع شدہ جوابات کی ڈیفالٹ ویلیو 1 ہوتی ہے، یعنی جیسے ہی کسی ریکارڈ کے پاس 1 جمع شدہ جواب ہوتا ہے اسے مکمل سمجھا جائے گا اور آپ کے dataset کی پیش رفت میں شمار ہوگا۔

بعض اوقات، آپ ہر ریکارڈ کے لیے ایک سے زیادہ جمع شدہ جوابات چاہتے ہیں، مثال کے طور پر، اگر آپ اپنے ٹاسک میں inter-annotator agreement کا تجزیہ کرنا چاہتے ہیں۔ اس صورت میں، اس سیٹنگ کو زیادہ تعداد میں تبدیل کرنا یقینی بنائیں، لیکن ہمیشہ کل annotators کی تعداد کے برابر یا کم۔ اگر آپ اکیلے کام کر رہے ہیں تو آپ چاہتے ہیں کہ یہ سیٹنگ 1 ہی رہے۔

## ریکارڈز کو annotate کریں

>[!TIP]
>💡 اگر آپ Argilla کو Hugging Face Space میں deploy کر رہے ہیں، تو آپ کے کسی بھی ٹیم ممبر کو Hugging Face OAuth کا استعمال کرتے ہوئے لاگ ان ہونے کی اجازت ہوگی۔ ورنہ، آپ کو ان کے لیے صارف بنانے کی ضرورت پڑ سکتی ہے، جس کے لیے [اس guide](https://docs.argilla.io/latest/how_to_guides/user/) کی پیروی کریں۔

جب آپ اپنا dataset کھولتے ہیں، تو آپ کو معلوم ہو جائے گا کہ پہلا سوال پہلے ہی کچھ تجویز کردہ labels کے ساتھ پر ہے۔ ایسا اس لیے ہے کیونکہ پچھلے حصے میں ہم نے اپنے سوال جس کا نام `label` ہے کو dataset کے `label_text` کالم سے map کر دیا تھا، تاکہ ہمیں صرف پہلے سے موجود labels کا جائزہ لے کر درست کرنا ہو۔

<img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter10/argilla_initial_dataset.png" alt="Screenshot of the dataset in Argilla."/>

token classification کے لیے، ہمیں تمام labels کو دستی طور پر شامل کرنے کی ضرورت ہوگی، کیونکہ ہم نے کوئی تجویز شامل نہیں کی۔ یہ span annotations کے بعد کچھ اس طرح نظر آ سکتا ہے:

<img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter10/argilla_dataset_with_spans.png" alt="Screenshot of the dataset in Argilla with spans annotated."/>

جیسا کہ آپ مختلف ریکارڈز سے گزرتے ہیں، آپ مختلف اقدامات کر سکتے ہیں: 
- جب آپ ریکارڈ کے ساتھ کام مکمل کر لیں تو اپنے جوابات جمع کر دیں۔
- اگر آپ بعد میں واپس آنا چاہتے ہیں تو انہیں ڈرافٹ کے طور پر محفوظ کر لیں۔
- اگر ریکارڈ dataset کا حصہ نہیں ہونا چاہیے یا آپ اس پر جوابات نہیں دینا چاہتے تو انہیں مسترد کر دیں۔

اگلے حصے میں، آپ سیکھیں گے کہ آپ ان annotations کو کیسے export کر کے استعمال کر سکتے ہیں۔
