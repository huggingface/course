## 突破性的“Aha”时刻

![“Aha”时刻](https://huggingface.co/reasoning-course/images/resolve/main/grpo/9.png)

R1-Zero训练过程中最引人注目的发现之一，是出现了一种被称为“Aha Moment”（顿悟时刻）的现象。这种现象与人类在解决问题是时突然“开窍”的过程颇为相似。具体运行机制如下：

1. 初始尝试：模型尝试给出问题的初始解。
2. 识别反思：模型识别出潜在的错误或逻辑不自洽之处。
3. 自我纠正：基于上述发现，模型自主调整解题思路。
4. 原理解释：模型能够解释为什么新的方法更优。

这个突破让学习者感同身受，仿佛经历了一次“尤里卡”式的灵光乍现时刻。它证明了模型是在真正“学习”，而不仅仅是“死记硬背”。现在，让我们来一起想象一下，茅塞顿开的“Aha”时刻究竟是一种什么样的体验。

举个例子，想象一下你正在拼拼图🧩：
- 初步尝试：“根据颜色判断，这一块应该放在这里”
- 发现问题：“但是等一下，这个形状好像不能完全贴合。”
- 自我纠正：“啊，它其实应该放在那边。”
- 原理解释：“因为只有颜色和形状都吻合，才能把这一块拼到这个位置上。”

这种能力是在强化学习训练中自然涌现的，而非通过显式编程设定。它证明了模型是在真正地“学习”，而不仅仅是单纯地从训练数据中记忆解题过程。

## 训练过程

训练R1是一个多阶段的过程，让我们拆解各个阶段及其核心创新。

这一流程最终产出了两个模型：
- DeepSeek-R1-Zero：完全仅通过强化学习训练而成的模型
- DeepSeek-R1：在DeepSeek-R1-Zero的基础上，引入了有监督微调（SFT）进行迭代的模型

| 特征 | DeepSeek-R1-Zero | DeepSeek-R1 |
|---------|------------------|--------------|
| 训练方法 | 纯强化学习 | 多阶段训练（有监督微调 + 强化学习） |
| 微调 | 无 | 有监督微调 |
| 推理能力 | 涌现 | 加强 |
| AIME效果 | 71.0% | 79.8% |
| 关键特点 | 强推理能力，但存在可读性问题 | 更好的语言连贯性和可读性 |

虽然DeepSeek-R1-Zero证明了纯强化学习在培养模型的推理能力方面的巨大潜力，但DeepSeek-R1在此基础上，采取了更均衡的训练方法，兼顾推理性能和模型实用性。

DeepSeek-R1的训练过程包含以下四个阶段：

1. 冷启动阶段
2. 推理强化学习阶段
3. 拒绝采样阶段
4. 发散强化学习阶段

让我们深入梳理每一个阶段：

### 冷启动阶段（模型质量的基础）

![冷启动阶段](https://huggingface.co/reasoning-course/images/resolve/main/grpo/5.png)

这个阶段是为了给模型的可读性和回复质量打下扎实的基础。研发团队以DeepSeek-V3-Base为起点，利用从R1-Zero中提取并经过验证的数千条高质量样本进行有监督微调（SFT）。这种创新的方法通过使用“小而精”的数据集， 成功为模型构建了强大的初始可读性与高质量回复的基准。

### 推理强化学习阶段（能力构建）

![推理强化学习阶段](https://huggingface.co/reasoning-course/images/resolve/main/grpo/6.png)

推理强化学习阶段侧重于培养跨领域的深度推理能力，涵盖数学、编程、科学及逻辑等学科。这个阶段采用了基于规则的强化学习，奖励的设计也直接和解题正确性挂钩。

至关重要的一点是，在这个阶段的所有任务均具备“可验证性”。这意味着我们能够明确判断模型的答案是否正确。例如在处理数学题上，我们能够通过使用数学求解器（Solver）来自动验证结果。

这个阶段的创新之处在于其直接优化的方法————它省去了独立的奖励模型，从而极大地精简了训练。

### 拒绝采样阶段（质量控制）

![拒绝采样阶段](https://huggingface.co/reasoning-course/images/resolve/main/grpo/7.png)

在拒绝采样阶段，模型生成的样本会经过严格的质量控制流程筛选。DeepSeek-V3 担任质量评判者，在远超纯推理任务的更广泛范围内评估输出结果。经过筛选的高质量数据随后被用于有监督微调（SFT）。本阶段的创新之处在于，它能够整合多种质量信号，从而确保输出内容达到高标准。

### 发散强化学习阶段（宽度对齐）

![发散强化学习阶段](https://huggingface.co/reasoning-course/images/resolve/main/grpo/8.png)

最后的发散强化学习阶段会采用更精心设计的混合方案来应对不同种类的任务。对于确定性任务，系统采用基于规则的奖励机制；但对于主观性任务，则通过大模型的反馈进行评估。这个阶段通过创新的混合奖励机制实现人类偏好对齐，将规则系统的精确性和语言模型评估的灵活性完美结合。

## 算法：群体相对策略优化（GRPO）

现在我们已经深入了解了模型的训练流程，接下来让我们看看用于训练该模型的具体算法。

作者们将GRPO描述为模型微调领域的一项重大突破：

![GRPO流程图](https://huggingface.co/reasoning-course/images/resolve/main/grpo/10.png)

GRPO的新颖之处在于它能够“直接针对偏好修正进行优化”。与PPO等传统强化学习算法相比, 它为模型与预期输出的对齐指明了一条更直接、高效的路径。让我们通过它的三个核心组件来拆解GRPO是怎么工作的。

### 分组：生成多种解决方案

GRPO的第一步非常地直观 —— 它就像学生在面对难题时，会尝试通过多种路径来寻求答案。当接收到提示词时，模型不会只生成一个答案；相反，模型会针对同一个问题产生多个解题尝试（通常是4、8或16个不同的版本）。

想象一下，你正在教一个模型解决数学题。当问题是计算农场里有多少只鸡时，模型可能会生成几个不一样的解决方法：
- 解法一是把问题拆解成不同步骤：先计算鸡的总数，再减去公鸡的数量，最后计算出不下蛋的母鸡的数量。
- 解法二可能会采用另一种截然不同但同样行之有效的方法。
- 其他尝试中则可能包含错误或效率较低的步骤。

所有这些尝试都会被整合为一个“组”（Group），就像把多名学生的试卷放在一起进行对比和学习。

![分组](https://huggingface.co/reasoning-course/images/resolve/main/grpo/11.jpg)

### 偏好学习：理解什么是一个优质解决方案

GRPO的魅力在于其偏好学习机制极其简洁。与传统的RLHF方法（通常需要一个独立的奖励模型来预测方案好坏）不同，GRPO可以调用任意函数或模型去评估方案的质量。例如，我们可以利用长度函数去鼓励更简洁的回复，或者通过数学求解器去奖励计算正确的解法。

评估过程会从多个维度审视每一个方案：
- 最终的答案是否正确？
- 解题格式是否符合规范（例如使用正确的XML标签）？
- 推理过程与最终答案是否逻辑一致？

这种方法的巧妙之处在于其评分机制：GRPO 并不直接使用绝对评分，而是对每个组内的奖励值进行归一化（Normalize）处理。它采用了一个简单而有效的公式来计算群组相对优势（Group Relative Advantage）：

```
Advantage = (reward - mean(group_rewards)) / std(group_rewards)
```

![偏好学习](https://huggingface.co/reasoning-course/images/resolve/main/grpo/12.jpg)

这种标准化处理就像是 AI 界的“按排名给分”。它能帮助模型理解组内哪些方案优于或劣于同类尝试，而不仅仅是盯着孤立的绝对分数。

### 优化：从经验中学习

最后一步是GRPO算法基于组内方案的评估结果，引导模型进行自我进化。这个过程运用了以下两个大原则，使得训练兼具高效性与稳定性：

1. 鼓励模型生成类似于成功案例的方案，同时抑制那些低效或错误的解题路径。
2. 引入 KL 散度惩罚作为约束，防止模型的策略更新过于激进。

相比传统方法，GRPO 已被证明具有更高的稳定性，原因在于：
- 它通过同时审视多个解题方案来进行学习，而非局限于两两对比。
- 基于组的标准化可以有效缓解奖励缩放（Reward Scaling）带来的训练不稳定问题。
- KL惩罚就像一个安全网，它保证模型在学习新技能的同时，不会偏离已有的知识储备。

> [!TIP]
> GRPO的核心创新：
> - 灵活的评价源：可直接从任意函数或模型中学习，彻底摆脱了了对独立奖励模型（Reward Model）的依赖。
> - 群体化学习：相比传统的两两比对法（Pairwise Comparison），基于组的学习模式更加稳定和高效。

尽管这些技术细节较为复杂，但最核心的结论是：GRPO 为训练模型具备推理能力提供了一条更高效、更稳定的技术路径。

### GRPO算法的伪代码

现在我们已经了解了GRPO的核心组成部分，让我们来看看算法的伪代码。这是一个简化版的算法，但关键点都囊括其中。

```
输入：
- initial_policy：待训练的当前策略模型
- reward_function：输出结果评价函数（奖励函数）
- training_prompts：训练数据集
- group_size：每个提示词生成的样本数 (G，通常为 4-16)

GRPO算法流程：
1. 每一轮训练迭代：
   a. 令 reference_policy = initial_policy（创建当前策略的快照作为参考）
   b. 对于批处理（Batch）中的每个提示词：
      i. 使用 initial_policy 生成 G 个不同的输出方案
      ii. 使用 reward_function 计算每个方案的原始奖励值
      iii. 在组内对奖励值进行归一化处理：
           归一化优势 (Advantage) = (当前奖励 - 组内平均奖励) / 组内标准差
      iv. 通过最大化“裁剪后的目标函数”来更新策略：
          min(概率比 * 归一化优势, 
               clip(概率比, 1-ε, 1+ε) * 归一化优势)
           - KL 权重 * KL 散度(当前策略 || 参考策略)
           其中，概率比 = 当前概率 / 参考概率

输出：优化后的策略模型
```

这段伪代码直观地展示了 GRPO 的运行逻辑：它巧妙地将“组内相对优势估计”整合进策略优化过程，并辅以“裁剪机制（Clipping）”与“KL 散度约束”，从而在大幅提升训练效率的同时，确保了模型更新的稳健性。

## GRPO的局限与挑战

虽然GRPO代表了强化学习在大语言模型上的重要进展，但是了解它的局限和挑战仍然非常重要。

- **生成成本：** 与仅需生成一两个结果的方法相比，GRPO 针对每个提示词生成多个候选输出（4-16个）的特性，显著提升了算力开销。
- **批次大小（Batch Size）的限制**：由于需要同时处理整组输出，这限制了有效批次大小的规模，增加了训练流程的复杂性，并可能降低训练速度。
- **奖励函数设计**：训练质量高度依赖于奖励函数的优劣。设计不当的奖励机制可能导致模型产生非预期行为，甚至导致优化目标发生偏移。
- **组大小的权衡**：选择最优组大小需在解空间的多样性与计算成本之间寻找平衡。样本过少会导致缺乏足够的多样性，而样本过多则会剧增训练时间与资源消耗。
- **KL散度调优**： 寻找 KL 散度惩罚的平衡点需要精细的调优————权重过高会抑制模型的学习效率，权重过低则可能导致模型偏离初始策略过远，从而丧失原有能力。

## 结论

DeepSeek R1 论文是大语言模型发展史上的一座重要里程碑。群组相对策略优化（GRPO）算法有力地证明了，纯强化学习确实能独立催生出强大的推理能力。这一发现打破了业界此前认为“有监督微调（SFT）是必经之路”的固有认知。

或许更具意义的是，DeepSeek R1 证明了卓越性能与经济实用性、低门槛化是可以兼得的。通过将模型能力成功蒸馏至 1.5B 到 70B 等不同参数规模，DeepSeek 为推动前沿 AI 技术的普及开辟了一条清晰的道路。
