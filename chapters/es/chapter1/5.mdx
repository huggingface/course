En [Transformers, ¿qué pueden hacer?](/course/chapter1/3) viste tareas de procesamiento de lenguaje natural, audio y visión por computador. En esta página veremos con más detalle cómo los modelos resuelven esas tareas y qué sucede internamente. Aunque existen muchas formas de abordar una tarea, en los modelos Transformer la idea general suele ser la misma: la mayoría son variantes de un encoder, un decoder o una arquitectura encoder-decoder.

> [!TIP]
> Antes de entrar en variantes arquitectónicas concretas, conviene recordar que la mayoría de tareas siguen un patrón parecido: los datos de entrada pasan por un modelo y la salida se interpreta para una tarea específica. Lo que cambia es cómo se preparan los datos, qué variante arquitectónica se usa y cómo se procesa la salida.

Para explicar cómo se resuelven las tareas, repasaremos qué ocurre dentro del modelo para producir predicciones útiles. Cubriremos los siguientes modelos y tareas asociadas:

- [DETR](https://huggingface.co/docs/transformers/model_doc/detr) para detección de objetos.
- [Mask2Former](https://huggingface.co/docs/transformers/model_doc/mask2former) para segmentación de imágenes.
- [GLPN](https://huggingface.co/docs/transformers/model_doc/glpn) para estimación de profundidad.
- [BERT](https://huggingface.co/docs/transformers/model_doc/bert) para tareas de PLN como clasificación de texto, clasificación de tokens y question answering.
- [GPT-2](https://huggingface.co/docs/transformers/model_doc/gpt2) para generación de texto.
- [BART](https://huggingface.co/docs/transformers/model_doc/bart) para tareas como resumen y traducción.

## Modelos Transformer para lenguaje

Los modelos del lenguaje están en el corazón del PLN moderno. Están diseñados para entender y generar lenguaje humano aprendiendo patrones estadísticos y relaciones entre palabras o tokens.

El Transformer se diseñó inicialmente para traducción automática y desde entonces se ha convertido en la arquitectura por defecto para resolver tareas de IA. Algunas tareas se adaptan mejor a la estructura encoder, otras al decoder y otras a una combinación de ambas.

### Cómo funcionan los modelos del lenguaje

Los modelos del lenguaje se entrenan para predecir la probabilidad de una palabra dado el contexto que la rodea. Eso les da una comprensión básica del lenguaje que luego puede reutilizarse en otras tareas.

Existen dos enfoques principales para entrenar un Transformer:

1. **Masked language modeling (MLM)**: lo usan modelos encoder como BERT. Se enmascaran algunos tokens y el modelo aprende a reconstruirlos a partir del contexto de ambos lados.
2. **Causal language modeling (CLM)**: lo usan modelos decoder como GPT. El modelo predice el siguiente token a partir de los anteriores de la secuencia.

### Tipos de modelos del lenguaje

En la librería Transformers, los modelos del lenguaje suelen caer en tres categorías arquitectónicas:

1. **Modelos solo encoder** como BERT: entienden bien el contexto bidireccional y son ideales para clasificación, reconocimiento de entidades y question answering extractivo.
2. **Modelos solo decoder** como GPT o Llama: procesan texto de izquierda a derecha y destacan en tareas de generación.
3. **Modelos encoder-decoder** como T5 o BART: combinan ambas aproximaciones y sobresalen en tareas sequence-to-sequence como traducción y resumen.

![transformer-models-for-language](https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter1/transformers_architecture.png)

Como vimos antes, estos modelos suelen preentrenarse sobre grandes cantidades de texto sin anotaciones humanas y luego se ajustan para tareas específicas. Este enfoque, conocido como transfer learning, permite adaptar un mismo modelo a muchas tareas con cantidades relativamente pequeñas de datos.

### Generación de texto

La generación de texto consiste en crear texto coherente y contextualmente relevante a partir de un prompt.

[GPT-2](https://huggingface.co/docs/transformers/model_doc/gpt2) es un modelo solo decoder preentrenado sobre una gran cantidad de texto. Puede generar texto convincente a partir de un prompt y también resolver algunas otras tareas de PLN.

<div class="flex justify-center">
    <img src="https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/gpt2_architecture.png"/>
</div>

GPT-2 usa byte pair encoding para tokenizar, añade codificaciones posicionales y pasa la secuencia por varios bloques decoder. Dentro de cada bloque, usa *masked self-attention*, lo que significa que no puede atender a tokens futuros. La salida del decoder se pasa a una language modeling head que transforma los hidden states en logits para predecir el siguiente token.

### Clasificación de texto

La clasificación de texto asigna categorías predefinidas a documentos, como análisis de sentimiento, clasificación temática o detección de spam.

[BERT](https://huggingface.co/docs/transformers/model_doc/bert) es un modelo solo encoder y fue el primero en implementar con éxito una bidireccionalidad profunda para aprender representaciones más ricas del texto atendiendo a palabras a ambos lados.

BERT usa tokenización WordPiece, añade tokens especiales como `[CLS]` y `[SEP]`, y se preentrena con dos objetivos: masked language modeling y next-sentence prediction. Para usarlo en clasificación de texto se añade una cabeza de clasificación sobre el modelo base.

### Clasificación de tokens

La clasificación de tokens consiste en asignar una etiqueta a cada token de una secuencia, como en named entity recognition o part-of-speech tagging.

Para usar BERT en este tipo de tareas, se añade una cabeza de token classification sobre el modelo base, que transforma los hidden states finales en logits para cada token.

### Question answering

Question answering consiste en encontrar la respuesta a una pregunta dentro de un contexto dado.

Para usar BERT en question answering, se añade una cabeza de clasificación de spans que calcula logits de inicio y fin para localizar el fragmento de texto correspondiente a la respuesta.

### Resumen automático

El resumen consiste en condensar un texto largo en una versión más corta conservando su información principal.

Los modelos encoder-decoder como [BART](https://huggingface.co/docs/transformers/model_doc/bart) y T5 están diseñados para el patrón sequence-to-sequence que requiere esta tarea.

<div class="flex justify-center">
    <img src="https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/bart_architecture.png"/>
</div>

BART tiene un encoder muy similar a BERT y se preentrena corrompiendo la entrada y reconstruyéndola con el decoder. Una estrategia clave es *text infilling*, donde varios fragmentos se sustituyen por un único token de máscara.

### Traducción

La traducción consiste en convertir texto de un idioma a otro preservando el significado. También es una tarea sequence-to-sequence, por lo que modelos encoder-decoder como BART o T5 son adecuados.

BART puede adaptarse a traducción añadiendo un encoder específico para el idioma fuente y utilizando el decoder para generar el idioma objetivo. Más adelante apareció mBART, una versión multilingüe pensada explícitamente para traducción.

## Modalidades más allá del texto

Los Transformers no se limitan al texto. También pueden aplicarse a voz, audio, imágenes y video. En este curso nos centraremos en texto, pero conviene tener una visión general de otras modalidades.

### Voz y audio

[Whisper](https://huggingface.co/docs/transformers/main/en/model_doc/whisper) es un Transformer encoder-decoder preentrenado sobre 680.000 horas de audio etiquetado. Gracias a esa escala, logra buen rendimiento zero-shot en tareas de audio en inglés y muchos otros idiomas.

<div class="flex justify-center">
    <img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter1/whisper_architecture.png"/>
</div>

Whisper tiene dos componentes principales:

1. Un **encoder** procesa el audio de entrada, que primero se convierte en un log-Mel spectrogram.
2. Un **decoder** toma la representación codificada y genera autorregresivamente los tokens de texto correspondientes.

### Reconocimiento automático de voz

Para reconocimiento automático de voz se usa la estructura completa encoder-decoder. El encoder procesa el audio y el decoder genera la transcripción token a token. La manera más sencilla de hacer inferencia es usar un `pipeline`.

```python
from transformers import pipeline

transcriber = pipeline(
    task="automatic-speech-recognition", model="openai/whisper-base.en"
)
transcriber("https://huggingface.co/datasets/Narsil/asr_dummy/resolve/main/mlk.flac")
```

### Visión por computador

Hay dos maneras principales de abordar tareas de visión por computador:

1. Dividir una imagen en una secuencia de parches y procesarlos en paralelo con un Transformer.
2. Usar una CNN moderna, como [ConvNeXT](https://huggingface.co/docs/transformers/model_doc/convnext), que sigue apoyándose en convoluciones pero adopta diseños modernos.

### Clasificación de imágenes

Tanto ViT como ConvNeXT pueden usarse para clasificación de imágenes. La diferencia principal es que ViT usa atención, mientras que ConvNeXT usa convoluciones.

[ViT](https://huggingface.co/docs/transformers/model_doc/vit) reemplaza por completo las convoluciones con una arquitectura Transformer pura.

<div class="flex justify-center">
    <img src="https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/transformers/model_doc/vit_architecture.jpg"/>
</div>

ViT divide la imagen en parches, convierte cada parche en un embedding, añade un token `[CLS]`, suma embeddings posicionales y pasa la secuencia al encoder Transformer. La salida asociada a `[CLS]` se usa para clasificar la imagen.
