# Evaluación

Con un modelo ajustado mediante SFT o LoRA SFT, deberíamos evaluarlo en benchmarks estándar. Como ingenieros de machine learning, debéis mantener un conjunto de evaluaciones relevantes para vuestro dominio objetivo.

## Benchmarks automáticos

Los benchmarks automáticos sirven como herramientas estandarizadas para evaluar modelos de lenguaje en distintas tareas y capacidades.

## Benchmarks de conocimiento general

[MMLU](https://huggingface.co/datasets/cais/mmlu) pone a prueba conocimientos en 57 materias. TruthfulQA evalúa la tendencia de un modelo a reproducir ideas equivocadas comunes.

## Benchmarks de razonamiento

[BBH](https://huggingface.co/datasets/lukaemon/bbh) y [GSM8K](https://huggingface.co/datasets/openai/gsm8k) se centran en tareas de razonamiento complejas.

## Comprensión del lenguaje

[HELM](https://github.com/stanford-crfm/helm) proporciona un marco de evaluación holístico.

## Benchmarks específicos de dominio

El benchmark MATH, HumanEval y Alpaca Eval cubren razonamiento matemático, generación de código y seguimiento de instrucciones, respectivamente.

## Enfoques alternativos de evaluación

### LLM-as-Judge
Usar un modelo de lenguaje para evaluar las salidas de otro se ha vuelto cada vez más popular.

### Arenas de evaluación
Las arenas de evaluación como [Chatbot Arena](https://lmarena.ai/) ofrecen un enfoque basado en comparaciones por pares y votos de usuarios.

### Suites de benchmarks personalizados
Las organizaciones suelen desarrollar suites internas adaptadas a sus casos de uso.

## Evaluación personalizada

1. Empezad con benchmarks estándar relevantes.
2. Identificad los requisitos y desafíos específicos de vuestro caso de uso.
3. Desarrollad conjuntos de datos de evaluación personalizados.
4. Considerad una estrategia de evaluación en varias capas.

## Implementar evaluaciones personalizadas

Podemos usar [`lighteval`](https://github.com/huggingface/lighteval) para evaluar nuestro modelo ajustado en benchmarks estándar.

Las tareas de LightEval se definen con el formato:

```
{suite}|{task}|{num_few_shot}|{auto_reduce}
```

## Ejemplo de pipeline de evaluación

```bash
lighteval accelerate \
    "pretrained=your-model-name" \
    "mmlu|anatomy|0|0" \
    "mmlu|high_school_biology|0|0" \
    "mmlu|high_school_chemistry|0|0" \
    "mmlu|professional_medicine|0|0" \
    --max_samples 40 \
    --batch_size 1 \
    --output_path "./results" \
    --save_generations true
```

> [!TIP]
> ✏️ **¡Pruébalo!** Evalúa tu modelo ajustado en una tarea específica de lighteval.
