மாடல் ஹப் பொருத்தமான மாதிரியைத் தேர்ந்தெடுப்பதை எளிதாக்குகிறது, எனவே எந்தவொரு கீழ்நிலை நூலகத்திலும் அதைப் பயன்படுத்துவது சில வரிக் குறியீட்டில் செய்யப்படலாம். இந்த மாதிரிகளில் ஒன்றை உண்மையில் எவ்வாறு பயன்படுத்துவது, மற்றும் சமூகத்திற்கு எவ்வாறு பங்களிப்பது என்பதைப் பார்ப்போம்.

முகமூடி நிரப்புதலைச் செய்யக்கூடிய பிரெஞ்சு அடிப்படையிலான மாதிரியை நாங்கள் தேடுகிறோம் என்று வைத்துக்கொள்வோம்.

<div class="flex justify-center">
<img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter4/camembert.gif" alt="Camembert மாதிரியைத் தேர்ந்தெடுக்கிறது." width="80%"/>
</div>

அதை முயற்சி செய்ய `camembert-base` சோதனைச் சாவடியை நாங்கள் தேர்ந்தெடுக்கிறோம். `camembert-base` என்ற அடையாளங்காட்டி அதைப் பயன்படுத்தத் தொடங்க நமக்குத் தேவையானது! முந்தைய அத்தியாயங்களில் நீங்கள் பார்த்தபடி, `pipeline()` செயல்பாட்டைப் பயன்படுத்தி அதை நாம் உருவாக்கலாம்:

```py
from transformers import pipeline

camembert_fill_mask = pipeline("fill-mask", model="camembert-base")
results = camembert_fill_mask("Le camembert est <mask> :)")
```

நீங்கள் பார்க்கிறபடி, ஒரு பைப்லைனில் ஒரு மாதிரியை ஏற்றுவது மிகவும் எளிது. நீங்கள் கவனிக்க வேண்டிய ஒரே விஷயம், தேர்ந்தெடுக்கப்பட்ட சோதனைச் சாவடி அது பயன்படுத்தப்படப் போகும் பணிக்கு ஏற்றது என்பதுதான். எடுத்துக்காட்டாக, இங்கே நாம் `camembert-base` சோதனைச் சாவடியை `fill-mask` பைப்லைனில் ஏற்றுகிறோம், இது முற்றிலும் சரி. ஆனால் இந்த சோதனைச் சாவடியை `text-classification` பைப்லைனில் ஏற்றினால், முடிவுகள் எந்த அர்த்தத்தையும் தராது, ஏனெனில் `camembert-base`-இன் தலை இந்த பணிக்கு ஏற்றது அல்ல! பொருத்தமான சோதனைச் சாவடிகளைத் தேர்ந்தெடுக்க ஹக்கிங் ஃபேஸ் ஹப் இடைமுகத்தில் உள்ள பணித் தேர்வியைப் பயன்படுத்த பரிந்துரைக்கிறோம்:

மாதிரி கட்டமைப்பைப் பயன்படுத்தி நேரடியாக சோதனைச் சாவடியை நீங்கள் உருவாக்கலாம்:

{#if fw === 'pt'}
```py
from transformers import CamembertTokenizer, CamembertForMaskedLM

tokenizer = CamembertTokenizer.from_pretrained("camembert-base")
model = CamembertForMaskedLM.from_pretrained("camembert-base")
```

இருப்பினும், [`Auto*` வகுப்புகளை](https://huggingface.co/transformers/model_doc/auto?highlight=auto#auto-classes)ப் பயன்படுத்த பரிந்துரைக்கிறோம், ஏனெனில் இவை வடிவமைப்பால் கட்டமைப்பு-அறியாதவை. முந்தைய குறியீடு மாதிரி பயனர்களை CamemBERT கட்டமைப்பில் ஏற்றக்கூடிய சோதனைச் சாவடிகளுக்கு மட்டுப்படுத்தும்போது, `Auto*` வகுப்புகளைப் பயன்படுத்துவது சோதனைச் சாவடிகளை மாற்றுவதை எளிதாக்குகிறது:

```py
from transformers import AutoTokenizer, AutoModelForMaskedLM

tokenizer = AutoTokenizer.from_pretrained("camembert-base")
model = AutoModelForMaskedLM.from_pretrained("camembert-base")
```
{:else}
```py
from transformers import CamembertTokenizer, TFCamembertForMaskedLM

tokenizer = CamembertTokenizer.from_pretrained("camembert-base")
model = TFCamembertForMaskedLM.from_pretrained("camembert-base")
```

இருப்பினும், [`TFAuto*` வகுப்புகளை](https://huggingface.co/transformers/model_doc/auto?highlight=auto#auto-classes)ப் பயன்படுத்த பரிந்துரைக்கிறோம், ஏனெனில் இவை வடிவமைப்பால் கட்டமைப்பு-அறியாதவை. முந்தைய குறியீடு மாதிரி பயனர்களை CamemBERT கட்டமைப்பில் ஏற்றக்கூடிய சோதனைச் சாவடிகளுக்கு மட்டுப்படுத்தும்போது, `TFAuto*` வகுப்புகளைப் பயன்படுத்துவது சோதனைச் சாவடிகளை மாற்றுவதை எளிதாக்குகிறது:

```py
from transformers import AutoTokenizer, TFAutoModelForMaskedLM

tokenizer = AutoTokenizer.from_pretrained("camembert-base")
model = TFAutoModelForMaskedLM.from_pretrained("camembert-base")
```
{/if}

<Tip>
ஒரு முன்பயிற்சி செய்யப்பட்ட மாதிரியைப் பயன்படுத்தும்போது, அது எவ்வாறு பயிற்சி செய்யப்பட்டது, எந்தத் தரவுத்தொகுப்புகளில், அதன் வரம்புகள் மற்றும் அதன் சார்புகளைச் சரிபார்க்கவும். இந்தத் தகவல்கள் அனைத்தும் அதன் மாதிரி அட்டையில் குறிப்பிடப்பட வேண்டும்.
</Tip>
